Feature selection – Ablation testing

To achieve optimal machine learning results and substantially improve the performance and robustness of classifier (this particularly holds for domain adaptation), the features individually and in interaction with other features should be monitored and tested systematically for each corpus and classifier.

We decided to support this task by implementing simple a feature selection algorithm testing all the possible combinations of feature sets and returning the feature vector leading to the best results.

More efficient techniques (such as stepwise regression or subset selection) should be implemented as the number of features increases.

How to use

Ablation testing is accessible through ELAC’s command line interface. To run ablation testing, choose the appropriate option in the main menu (option 6).

Note that you are prompted for the following parameters:
output file specifies the path to the file used as output for the results.

Input

Input for Ablation testing is determined by the PathToTestFiles setting in the configuration file.
Furthermore, the acr systems to be used can be specified there.

The results (precision, recall and f-score) for each feature combination and the best combination are written to the stated output file path.
