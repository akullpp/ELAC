ELAC: Ensemble Learning for ACR-Systems

Training

Posted on 30. Juli 2010 [Edit] Leave a comment (0)

Featuredependent Performance Measurement (Semeval)

We measured the performance of each ACR system based on our traning corpus semeval_train50, consisting of 50% of the data used as training material for Semeval2010 Task2 (see: http://stel.ub.edu/semeval2010-coref/systems).

We used MUC6 as evaluation standard, which leads to some disadvantages, especially for JavaRap.

Recall

Overall, it seems to be very obvious, that BART is the clear winner regardless of the feature categories used for evaluation. While BART scores really well in Coreferences with matching string representation, it tends to be very robust across all tested feature categories.

LingPipe is intended for references to named entities, as the peak for neType != None makes clear.

JavaRap doesn’t recognize coreference chains, but only sequent coreference pairs, while the anaphora has to be a pronoun.
The usage of MUC6 is clearly one explanation for the substandard results.

Precision

Looking at precision, BART’s focus on Recall is pretty obvious as there are many feature categories, where JavaRap and Lingpipe clearly outperform BART’s performance. (Note that the 1.0 precision value for JavaRap is not positive at all, as JavaRap only extracts coreferent pairs with a pronominal anaphora)

F-Score

All in all, F-Score mirrors the recall results, with LingPipe being the only system to beat BART with the feature category anaphoraPos!= PRP.
