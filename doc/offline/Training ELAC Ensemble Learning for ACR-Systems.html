ELAC: Ensemble Learning for ACR-Systems

Training

Posted on 31. Juli 2010 [Edit] Leave a comment (0)

Performance measurement

This class provides a single user interface to test and compare the performance of multiple ACR systems depending on specific feature clusters.

Furthermore, this class is used to get meta results about the performance of the used ACR systems depending on the underlying features. This information is used for our Machine Learning component later on.

How to choose the ACR systems to be used

In general, each class extending the abstract class de.uniheidelberg.cl.swp.testacr.Runner constitutes an ACR system and can be used for Performance Measurement.

To choose, which runner to use, specify them in the configuration file using the option ACR Systems. Make sure to specify all necessary options for your runner classes (such as paths to models), if required. Multiple runners are separated by semicolon (;).

If you run Performance Measurement without any further options, you will get the overall performance for each system on all training data. This is the common procedure to create a training file for the Machine Learning component.

How to use feature dependent evaluation

Let’s assume, you’d only want to compare the performance of all coreference pairs with certain features. To achieve this, you have to set one ore multiple feature filter. See the documentation for configuration options on a description of feature filter and possible values for each feature. Also note, that you can easily add new features.

Once a feature filter has been set, only Coreference Pairs matching the required feature values are used for evaluation, although all features are extracted for each pair.

What other options can (and should) be used

  • Evaluation method: We provide MUC6 evaluation as well as direct neighbors only. Consult the documentation for configuration options for further information.
  • Input and output paths and manipulation: Make sure to specifiy paths to the corpora which should be used for Performance Measurement (by using the appropriate configuration options). You can choose to only write true or false results in the output file as well as the path to the file itself in the (yes, you guessed right!) configuration.

How to run Performance Measurement

Performance Measurement is accessible through ELAC’s command line interface.

To run the whole trainings process, choose the appropriate option in the CLI (Training) – of course you made sure, that all configuration options had been set correctly, didn’t you? …

Where to find (and how to read) the output files

Output is written to the path specified in the configuration file. For all systems, one arff file is written, containing the training data for our Machine Learning Component.

Furthermore, for each ACR system one file containing all it’s results for all the input files will be generated.

The file name format (e.g. de.uniheidelberg.cl.swp.testacr.JavaRapRunner_31-07-10-12-43.txt) specifies the used runner class as well as a timestamp to prevent overriding old versions.

In the file itself, all relevant information for the performance measurement process are given.

  • Feature selectors: The feature filters used for the current run.
  • Feature vector: The list of features that are extracted for each Coreference Pair.
  • File section: For each file, the filename and results are printed and success or failure are indicated:
    • FN: False negative – not recognized
    • TP: Correctly classified
    • FP: Erroneously predicted
    • A feature vector constituting the Coreference Pair follows the success indication
    • File results: the result in numbers
  • Overall results: Summary stating the results across all files as well as precision, recall and F-score.
