ELAC: Ensemble Learning for ACR-Systems

Configuration

Posted on 30. Juli 2010 [Edit] Leave a comment (0)

General Information

The config.xml file found in the ./config folder of the project directory is the central place to make customizations to several parts of the ensemble learning process, like adjusting the file paths to corpora or managing ACR subsystems.
On this site you can find explanations of every parameter and several example values.

Corpus Paths

The first portion of the config file contains paths to the different kinds of corpora used by the program. Relative paths are already set, but can be changed to fit your needs.
The two following paths (namely the paths to training and testing corpora respectively) must be set in any case to have ELAC running:


<!-- Test/Train -->
<entry key="PathToTrainingFiles">../Data/semeval_train50/</entry>
<entry key="PathToTestFiles">../Data/semeval_test/</entry>

For development purposes this can be basicly extended as needed as long as it sticks to the given syntax and is refered to correctly from within the code. For tests on domain adaptation it could look like this:


<!-- Test/Train -->
<entry key="PathToTrainingFiles">../Data/semeval_train50/</entry>
<entry key="PathToTestFiles">../Data/semeval_test/</entry>
<entry key="PathToDomainFiles">../Data/semeval_domainMMAX/</entry>

ACR Systems

As ELAC is programmed to be easily expandable you can change or add the subsystems for anaphora and coreference resolution (ACR) in the config file. Of course, every single system has to be implemented in the code, but this config option defines which of the implemented systems are actually used. At the moment, this comprises BART, JavaRAP and LingPipe:


<!-- ACR-Systems -->
<entry key=>"Runner">JavaRapRunner;BARTRunner;LingpipeRunner</entry>

Lingpipe Chunker

As LingPipe needs a chunking model in order to work correctly ELAC provides a MUC6 trained chunker. This can be changed (e.g. to a statistical model) for testing and development purposes.


<!--Lingpipe Chunker -->
<entry key="LingPipeChunkgerModel">
./res/ne-en-news-muc6.AbstractCharLmRescoringChunker
</entry>

Feature Extraction

This central part of the config file is used in the processes of training, testing and evaluation.

Evaluation Method

The first entry defines the evaluation method which is usually set to MUC6 evaluation which takes into account the cross product of a coreference chain to form coreference pairs, but can be switched to use direct neighbors only.


<!-- Feature Extraction -->
<entry key="EvaluationMethod">MUC6</entry>

Possible values:

  • MUC6
  • DIRECTNEIGHBORSONLY

Feature Filter

This parameter selects a sub-set of all possible features to be extracted from the corpus. Note that every feature featured (haha!) in this list has to be specified with a certain value in its own parameter. A short description of the number and kinds of arguments are given in brackets (leave them out in the actual file!) and possible values are mentioned after each feature.

For a detailed desciption of each feature please refer to the JavaDocs of its respective class.


<entry key="FeatureFilter">isSubj;neType</entry>

<entry key="isSubj">true</entry>
<entry key="neType">other</entry>

Possible values:

  • isSubj (boolean): reference to the subject of a sentence.
    • true
    • false
  • neType (string[;string;...]): kind of named entity, the anaphora refers to.
    • organization
    • location
    • person
    • other
  • pronounCountSentence (integer;integer): number of competing pronouns in the same sentence.
    • minimum;maximum
  • anaphoraPos (string[;string;...]): Part of Speech tag of the anaphora.
    • prp
    • prp$
    • nnp
    • nn
    • nns
    • dt
    • else
  • antecedentPos (string[;string;...]): Part of Speech tag of the antecedent.
    • prp
    • prp$
    • nnp
    • nn
    • nns
    • dt
    • else
  • sentenceOffset (integer;integer): distance between anaphora and antecedent in sentences.
    • minimum;maximum
  • stringMatch (boolean): specifies whether anaphora and antecedent have identical string representations.
    • true
    • false

Machine Learning

After the training process Weka is used for machine learning. This part of the config gives the user the ability tweak basic parameters without having to mess with Weka itself.

Weka provides several classifiers that can be used to evaluate your results:


<entry key="classifier">J48</entry>

Possible values:

  • BAYES
  • J48
  • BAGGING (subclassifier needed)
  • ADABOOST (subclassifier needed)
  • NEARESTNEIGHBOR (simple linear classifier)
  • ZeroR (rule based classifier)

Some of the above mentioned classifiers also need a subclassifier, which is specified by the following statement:


<entry key="subclassifier">J48</entry>

Possible values:

  • BAYES
  • J48

Weka can further be used with different arguments:


<entry key="options">N 4 -R</entry>

Refer to Weka's documentation for all possible options.

Stacking

The last statement allows the activation of stacking for the evaluation process. Setting it to anything different from true or leaving it blank will disable stacking.

If stacking is enabled, the configuration option for subclassifier is used as source for the level-0 classifier, whereas classifier specifies the level-1 (meta) learner.


<entry key="stacking">true</entry>

Possible values:

  • true: enables stacking
  • false: disables stacking

Output

In this last segment of the config file everything concerning the output can be tweaked, from file paths to output method.

The first two lines are reponsible for the paths to write the output and a log file containing a documentation of all processes started aswell as potential error messages.


<!-- Output -->
<entry key="PathForLog">../Data/entry>
<entry key="ResultOutputDir">../Data/entry>

The last two lines trigger the option of printing correctly (true positives) or incorrectly (false positives, false negatives) identified coreference pairs into the output file.


<entry key="PrintCorrectCorefPairs">false</entry>
<entry key="PrintWrongCorefPairs">false</entry>

Possible values:

  • true: enables the specified output
  • false: disables the specified output
