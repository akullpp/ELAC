ELAC: Ensemble Learning for ACR-Systems

Architecture

Posted on 3. August 2010 [Edit] Leave a comment (0)

Architectural Overview

ELAC consists of several different modules that add up to a fully working anaphora and coreference resolution system based on ensemble learning. This page intends to give you a complete overview over the different modules and how they work together, for details refer to their dedicated pages respectively.
Note that the ELAC package also comes with several standalone tools that were created during development and are described on their own pages, but not mentioned here any further.


Architectural overview

Configuration

The configuration file provides user/system dependant information for almost every module, from file paths to feature selection. Detailed information can be found here.

Input

The input corpora have to be annotated in MMAX2 format to be read by ELAC’s MMAX2 parser for further processing.

ACR Systems

ELAC uses ensemble learning, i.e. it makes use of several anaphora and coreference resolution (ACR) systems. At the moment this comprises BART, JavaRAP and LingPipe, each with its own advantages and disadvantages. But as ELAC provides easy to use interfaces, it is possible to extend this list in the future.
The ACR systems’ role in the process is described below.

Performance measurement

The ACR systems introduced above predict what they believe are coreference chains/pairs from the corpus using their own algorithms each. These hypothesis coreferences are then checked against a previously extracted correct list of coreferences (gold standard) and saved into an .arff file for later use.

Note that the coreference pair itself is not saved, but rather the feature vector which goes along with it, together with the name of the ACR system that provided it and the information whether this was indeed a coreference (+) or not (-).

Machine learning

The previously produced .arff file is given as input to Weka, a tool used for classification. Based on user options from the configuration file Weka uses different classifiers to decide which ACR system is to trust for which feature vector, thus improving the results of another check against the gold standard.

Output

After this last check against the gold standard precision, recall and both combined into the F-score are output as a degree for quality measurement.
